¿Por qué se crean los objetos?

- Para modelar cosas del mundo real es decir un objeto es un molde con una estructura definida (propiedades, métodos). Un objeto puede ser creado a partir de otro lo que se conoce como herencia.

¿Qué hubiera pasado si agregaba otra propiedad?

- Aqui entran a tallar términos como la abstracción y la herencia, la abstracción es crear un modelo de objeto lo mas simple posible apartir de algo complejo, es decir definir un modelo básico que contenga las propiedades y métodos mas comunes, luego si queremos agregar una propiedad debemos analizar si es un dato común o algo mas específico para definir si se agregará en este objeto ya existente o se crea otro modelo de objeto que herede la estructura del objeto base (propiedades y métodos) y se adicione esta nueva propiedad. Ejemplo tenemos el objeto persona que tiene la siguiente estructura { nombre, edad, sexo } y luego tendriamos el objeto docente que hereda del objeto persona tendría la siguiente estructura { nombre, edad, sexo, especialidad } como se ve las propiedades nombre, edad y sexo son heredados del objeto persona, y sólo la propiedad especialidad es única del objeto docente.

Persona { nombre, edad, sexo }

Docente { nombre, edad, sexo, especialidad }

¿Se puede generar de otra manera algún objeto?

- Se puede crear un objeto haciendo uso de una funcion normal en la cual se define un variable de tipo objeto vacio luego se le asignan las propiedades y el valor de sus atributos y métodos y se retorna la variable, o podemos usar las funciones constructoras que proporciona javascript en la cual ya no es necesario declarar ni retornar una variable de tipo objeto, sino que hacemos uso de la palabra reservada this. Hay que tener en cuenta que por convención el nombre de una función consutructora debe empezar con la primera letra en mayúscula para que sea identificada rápidamente y para hacer uso de ella se debe empezar con la palabra reservada new.
